# gen_gray_log_trans

## 函数签名

```rust
pub fn gen_gray_log_trans(src: &RgbImage) -> GrayImage
```

## 功能描述

对图像进行对数灰度变换，使用公式 `y = c · log(1 + x)` 重新映射像素值。对数变换能够扩展图像的暗部细节，压缩亮部范围，特别适合处理动态范围过大的图像（如傅里叶频谱）。

## 参数说明

- `src: &RgbImage` - 输入的 RGB 彩色图像引用

## 返回值

- `GrayImage` - 对数变换后的灰度图像

## 数学原理

对数变换的标准公式：

```
s = c · log(1 + r)
```

其中：
- `r` 是输入灰度值，范围 [0, 255]
- `s` 是输出灰度值
- `c` 是缩放常数，用于将输出映射到 [0, 255]
- `log` 是自然对数（ln）
- 加 1 是为了避免 log(0) 的未定义情况

### 缩放常数的计算

为了使最大输入值（255）映射到最大输出值（255）：

```
c = 255 / log(256)
```

推导：
- 当 `r = 255` 时，我们希望 `s = 255`
- 代入：`255 = c · log(1 + 255) = c · log(256)`
- 求解：`c = 255 / log(256) ≈ 45.986`

### 对数变换的特性

1. **非线性映射**：
   - 低灰度值被显著扩展
   - 高灰度值被压缩
   
2. **单调递增**：
   - 保持相对亮度顺序不变
   
3. **动态范围压缩**：
   - 输入范围 [0, 255] → 输出范围 [0, 255]
   - 但分布更均匀

示例映射：
- 输入 0 → 输出 0
- 输入 10 → 输出 ~111
- 输入 50 → 输出 ~181
- 输入 100 → 输出 ~212
- 输入 200 → 输出 ~245
- 输入 255 → 输出 255

## 实现逻辑

### 第 57-58 行：转换为灰度并计算缩放因子
```rust
let gray = to_grayscale(src);
let scale = 255.0 / 256_f32.ln();
```
**原理**：
- **第 57 行**：将 RGB 图像转换为灰度图
- **第 58 行**：计算缩放常数 `c`
  - `256_f32.ln()` 计算 ln(256) ≈ 5.5452
  - `scale = 255 / 5.5452 ≈ 45.986`

---

### 第 59 行：创建目标图像
```rust
let mut dst = GrayImage::new(gray.width(), gray.height());
```
**原理**：创建与输入相同尺寸的空白灰度图像。

---

### 第 60-66 行：遍历并应用对数变换
```rust
for y in 0..gray.height() {
    for x in 0..gray.width() {
        let value = gray.get_pixel(x, y)[0] as f32;
        let mapped = (value + 1.0).ln() * scale;
        dst.put_pixel(x, y, Luma([mapped.clamp(0.0, 255.0) as u8]));
    }
}
```
**原理**：

- **第 60-61 行**：双重循环遍历所有像素

- **第 62 行**：读取当前像素的灰度值并转换为浮点数

- **第 63 行**：应用对数变换公式
  - `value + 1.0`：将范围从 [0, 255] 转换为 [1, 256]
    - 避免 ln(0) 未定义
  - `.ln()`：计算自然对数
  - `* scale`：乘以缩放因子，映射到 [0, 255]
  
  **计算示例**：
  - 输入 `value = 50`
  - `(50 + 1).ln() = ln(51) ≈ 3.9318`
  - `3.9318 × 45.986 ≈ 180.8`

- **第 64 行**：限制范围并写入
  - `.clamp(0.0, 255.0)` 确保在合法范围内
  - `as u8` 转换为字节
  - 写入目标图像

---

### 第 67 行：返回结果
```rust
dst
```

## 应用场景

### 1. 傅里叶频谱显示
```rust
// 傅里叶变换的频谱值范围巨大（可能 0 到数百万）
// 对数变换可以将其压缩到可见范围
let spectrum_log = gen_gray_log_trans(&spectrum_img);
```

### 2. 暗部细节增强
```rust
// 夜间或低照度图像，暗部细节不可见
let enhanced = gen_gray_log_trans(&dark_img);
// 暗部被大幅提亮，细节变得可见
```

### 3. 医学影像处理
```rust
// X 光片、CT 扫描等动态范围大的图像
let medical_enhanced = gen_gray_log_trans(&xray_img);
```

## 与线性变换的对比

| 特性 | 线性变换 | 对数变换 |
|------|---------|---------|
| 公式 | `y = αx + β` | `y = c·ln(1+x)` |
| 映射关系 | 等比例缩放 | 非线性压缩 |
| 暗部处理 | 比例放大 | 显著扩展 |
| 亮部处理 | 比例放大 | 压缩 |
| 适用场景 | 对比度/亮度调整 | 动态范围压缩 |

## 逆变换

如果需要恢复原始值（近似）：

```
r = exp(s / c) - 1
```

## 性能说明

- 时间复杂度：O(W × H)
- 每像素操作：1 次加法、1 次对数运算、1 次乘法
- 对数运算相对耗时，但对于图像尺寸仍可接受
